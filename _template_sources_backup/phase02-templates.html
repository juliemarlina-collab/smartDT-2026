T05 Problem Statement
Create a clear Point of View problem statement from empathy evidence.
Not Started
A1
Interactive Activity
Review: look back at T02 Empathy Map, T03 Persona and T04 User Needs Summary.
Frame: write the user, need and insight before combining them.
Check: make sure the statement describes the problem only, not the solution.
Sample Problem Statement

User: Polytechnic students who buy lunch during short breaks.

Need: Need a faster and less stressful way to get lunch.

Insight: They avoid the canteen because long queues make them late for class.

POV Formula
[User] needs [Need] because [Insight].

Example: POLIPD students need a faster way to get lunch because long queues make them miss rest time and risk being late to class.
Smart Tip

A strong problem statement should be user-centred, research-backed, and solution-free.

Completion Checklist
T06 How Might We
Generate 5 HMW questions and choose the strongest one for ideation.
Not Started
A2
Interactive Activity
Start from T05: use your approved problem statement as the reference.
Open possibilities: HMW questions should invite many ideas, not one fixed solution.
Select: choose the question that is focused but still creative.
Sample HMW Questions

Problem Reference: Students struggle to get lunch quickly during short breaks.

  • How might we help students get lunch faster during peak hours?
  • How might we reduce canteen waiting stress for students?
  • How might we help students plan lunch before break time?
  • How might we make lunch access easier for students with tight schedules?
  • How might we improve the canteen experience without increasing student cost?
Good HMW Test

Not too broad: “How might we fix food?” Not too narrow: “How might we create a QR pre-order app?” The best HMW opens many solution possibilities.
