T01 Interview Guide
Prepare and conduct interviews with at least 3 real target users.
Not Started
A1
Interactive Activity
Before: decide the user group, location, and 5 open-ended questions.
During: listen actively, ask follow-up questions, and avoid suggesting answers.
After: record 3 key insights while the discussion is fresh.
Sample Interview Context

Target user: Student who often buys lunch at the canteen.

Good question: “Tell me about your experience during peak lunch time.”

  • Avoid yes/no questions.
  • Use prompts such as “Why?”, “Can you explain more?”, “What happened next?”
  • Observe behaviour, body language, and repeated complaints.

Smart Tip

Capture direct quotes. Real user words are useful later when writing your empathy map and persona.

T03 Persona Template
Create one realistic target user profile based on evidence.
Not Started
A3
Interactive Activity
Combine findings: use repeated patterns from multiple users.
Humanise the user: add a believable quote, clear goals, and real frustrations.
Sample Persona

Name: Aina, 20, Diploma student

Quote: “I need lunch to be fast because my break is short.”

Goals: Buy lunch quickly, avoid missing class, spend less.

Pain points: Long queue, limited choices, crowded space.

Smart Tip

Keep the persona specific, but evidence-based. If it sounds too general, revisit your empathy map and user needs summary.

Completion Checklist
T04 User Needs Summary
Summarise key findings, patterns and needs with evidence.
Not Started
A4
Interactive Activity
Look for patterns: identify repeated problems or needs across users.
Support with evidence: every need should link to quotes, behaviour, or observations.
Sample User Needs Summary

Pattern: Many students avoid the canteen during peak time.

Need: Students need a faster way to plan or buy lunch.

Evidence: 3 out of 4 users mentioned long queues and short break time.

Smart Tip

Write needs in a user-centred way. Example: “Students need a faster lunch process so they can return to class on time.”
